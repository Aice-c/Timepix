# 物理意义层面说明

这份文档解释“这个毕业论文项目在物理上要解决什么问题”。后续 agent 在改代码前应先读这里，否则容易只看到 CNN、ResNet、脚本和配置，却忽略这些代码服务的实验目标。

## 1. 研究目标

项目目标是：根据带电粒子打到 Timepix/Timepix3 探测器后形成的像素响应，识别粒子的入射极角。

这里的“极角”可以理解为粒子轨迹相对于探测器法线方向的倾斜程度。极角越接近垂直入射，轨迹在像素平面上的投影通常越短、更集中；极角越倾斜，轨迹在像素平面上的投影往往更长、更有方向性，像素激活形状和时间/能量分布也会发生变化。

当前主要任务是分类：每个样本属于一个离散角度类别，例如 `15`、`30`、`45`、`60`。代码里也有回归任务支持，但主线仍然是分类。

## 2. 数据的物理含义

每个样本是一个粒子事件在探测器局部区域内留下的二维像素阵列。当前训练代码按文本矩阵读取样本，常见尺寸是 `50 x 50`。

矩阵中每个元素对应一个像素：

- `0`：该像素没有被激活。
- 非 `0`：该像素被激活，数值表示该模态下的响应强度。

项目中涉及两个可能的模态：

- `ToT`：Time over Threshold。通常可近似理解为像素信号超过阈值的持续时间，和局部能量沉积/电荷量有关。
- `ToA`：Time of Arrival。表示像素到达时间相关信息，能反映轨迹不同位置的时间结构。

但两个粒子数据集的模态并不相同：

- alpha 粒子数据集有双模态：`ToT` 和 `ToA`。
- C/质子数据集只有 `ToT` 模态，没有 `ToA`。

这意味着对 alpha 可以研究单模态和双模态效果；对 C/质子不能直接使用 ToA，也不能配置 `['ToT', 'ToA']`，除非后续真的补充了对应 ToA 数据。

因此，一个粒子事件不是普通图片，而是一个带有物理含义的稀疏探测器响应图。

## 3. 为什么能用图像模型识别角度

带电粒子穿过探测器时，会沿路径激活一组像素。不同入射极角会影响：

- 激活像素数量。
- 轨迹投影长度。
- 轨迹长宽比。
- ToT 总量和空间分布。
- ToA 的时间梯度或时间扩展。
- 轨迹是否更接近点状、短线状或长线状。

这些信息天然是二维空间结构，所以可以把 ToT/ToA 矩阵作为 CNN 输入。CNN 学到的卷积特征大致对应局部形态、方向、长度、稀疏结构和模态间关联。

## 4. 单模态与多模态

代码支持只使用一种模态，也支持同时使用 `ToT + ToA`，但这要受数据集本身的模态限制。

单模态：

- `['ToT']`：只看能量/电荷相关分布。
- `['ToA']`：只看时间相关分布。

多模态：

- `['ToT', 'ToA']`：把两个二维矩阵按通道拼接，类似一张两通道图像。

多模态的物理动机是：ToT 和 ToA 描述同一粒子事件的不同方面。ToT 更偏能量沉积，ToA 更偏时间结构，二者结合可能更好地区分相近角度。

需要特别注意：这个多模态假设目前只适用于 alpha 数据集。C/质子数据集只有 ToT，因此 C/质子实验应视为 ToT 单模态角度识别任务。

## 5. 手工特征的物理意义

除了 CNN 自动提取特征，代码还支持手工特征。当前注册的手工特征主要是：

- `total_energy`：对某个模态矩阵求和。

对于 ToT，`total_energy` 可以作为总能量沉积或总信号强度的近似表征。对于 ToA，直接求和的物理解释要更谨慎，因为 ToA 是时间相关量，求和未必总有清晰物理含义。

代码会把手工特征与 CNN 中间特征拼接，再送入分类头。这相当于告诉模型：除了自己从图像里学，也可以直接参考一些有物理启发的全局量。

## 6. 类别有序性

角度类别不是普通无序类别。把 `15` 预测成 `30`，和把 `15` 预测成 `60`，物理误差显然不同。

代码中加入了 EMD/Wasserstein 风格损失和角度 MAE 指标，就是为了利用角度类别的有序性：

- CrossEntropy 只关心类别是否正确。
- EMD 更关心预测分布离真实角度有多远。
- MAE 直接用角度单位衡量误差，更适合论文解释。

## 7. 数据增强的物理含义

当前代码支持 90 度旋转增强。

这在物理上隐含一个假设：极角分类不应该依赖轨迹在探测器平面内的方位角。也就是说，同样倾斜程度的轨迹，不管朝平面内哪个方向延伸，极角标签应相同。

这个假设通常合理，但仍需注意：

- 如果探测器响应存在方向性非均匀，旋转增强可能引入偏差。
- 如果数据采集几何导致某些方位角本身有意义，旋转增强需要重新评估。

## 8. 当前论文主线

从代码和项目描述看，论文可以组织成这条主线：

1. Timepix/Timepix3 探测器采集带电粒子事件。
2. 每个事件得到对应模态的像素阵列：alpha 为 ToT/ToA 双模态，C/质子为 ToT 单模态。
3. 根据入射角文件夹生成监督标签。
4. 使用 CNN/ResNet 提取轨迹形态特征。
5. 尝试多模态输入和手工物理特征拼接。
6. 用分类准确率、混淆矩阵、角度 MAE 等指标评价模型。
7. 通过消融实验比较：
   - ResNet vs shallow model。
   - CrossEntropy vs EMD。
   - 分类 vs 回归。
   - 单模态 vs 多模态/手工特征。

## 9. 后续 agent 应避免的误解

- 不要把输入当成普通自然图像；它是稀疏探测器物理响应。
- 不要只看 accuracy；相邻角度误判和远距离角度误判物理意义不同。
- 不要随意改变归一化、log1p、裁剪和旋转策略，这些都会改变物理量的表达。
- 不要默认所有手工特征都有清晰物理意义；ToT 总和相对直观，ToA 总和需要谨慎解释。
- 不要把 `alpha` 和 `C/proton` 数据混在一起训练，除非实验设计明确要求跨粒子泛化或联合训练。
- 不要给 C/质子数据配置 `['ToT', 'ToA']`；它目前没有 ToA 数据。
